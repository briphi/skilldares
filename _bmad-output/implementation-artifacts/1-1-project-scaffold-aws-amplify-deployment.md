### Vite Scaffold — Why the Temp-Dir + Merge Approach

The Vite CLI (`npm create vite@latest`) refuses to scaffold into a non-empty directory unless you pass `--overwrite`. Using `--overwrite` in this repo is risky because Vite's idea of "empty" doesn't account for hidden folders like `.git/`, `.claude/`, etc. — and the safety semantics across CLI versions have been inconsistent.

**Safer pattern (described in Task 2):**
1. Scaffold into a temporary directory OUTSIDE the repo (e.g., `~/skilldares-app-temp/`)
2. Copy the scaffolded files into this repo's root, file-by-file
3. Merge `.gitignore` rather than overwrite
4. Run `npm install` in the repo root

This avoids any risk to existing project artifacts.

### AWS Amplify Configuration

**`amplify.yml` exact content (commit to repo root):**

```yaml
version: 1
frontend:
  phases:
    preBuild:
      commands:
        - npm ci
    build:
      commands:
        - npm run build
  artifacts:
    baseDirectory: dist
    files:
      - '**/*'
  cache:
    paths:
      - node_modules/**/*
```

**Why these specific settings:**
- `npm ci` (instead of `npm install`) in preBuild → exact `package-lock.json` install, faster + deterministic builds on Amplify
- `npm run build` → Vite's production build, outputs to `dist/`
- `baseDirectory: dist` → tells Amplify where to find the static files to serve
- `node_modules/**/*` cache → speeds up subsequent builds significantly

**AWS Amplify Console setup is a manual user action** — Task 7 surfaces the steps. Once the Amplify app is connected to the GitHub repo + `main` branch, every push triggers an auto-deploy.
